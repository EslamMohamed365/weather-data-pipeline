## When to Reference

Reference these documents when:
- **Debugging**: Detailed error analysis and solutions
- **Performance Tuning**: In-depth profiling data and analysis
- **Historical Context**: Understanding design decisions and evolution
- **Code Review**: Detailed quality analysis and recommendations
- **Security Audit**: Complete security review and fix documentation

## New Documentation Structure

The new documentation is optimized for portfolio presentation and developer onboarding:

- **README.md** - Portfolio hero page (2-3 min read)
- **docs/SETUP.md** - Complete setup guide
- **docs/ARCHITECTURE.md** - Technical deep-dive
- **docs/PERFORMANCE.md** - Performance analysis
- **docs/API.md** - Developer reference
- **CONTRIBUTING.md** - Contribution guidelines
